LU-0000 lov: recognize SOM pattern in lov_entry_type

Add LOV_PATTERN_SOM check to lov_entry_type() so that SOM
components are recognized as valid layout entries instead
of being treated as unknown patterns.

SOM pattern (0x1000) is a standalone pattern, not combined
with RAID0 like parity patterns. Update the switch statement
in lov_init_composite() to handle LOV_PATTERN_SOM and assign
raid0_ops to it.

This fixes the "unknown layout entry pattern 0x1000" warning
that appeared when creating SOM mirrors.
