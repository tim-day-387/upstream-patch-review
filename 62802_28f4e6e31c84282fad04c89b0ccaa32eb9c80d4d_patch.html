LU-0000 lod: prevent SOM mirrors from being write targets

SOM (Size-on-MDS) mirrors store file size metadata on
the MDS and should never be selected as write targets.
They can only be updated through resync operations from
their corresponding data mirrors.

Add checks in mirror selection logic to skip SOM
mirrors when choosing a primary mirror for writes.

This patch adds:
- lod_mirror_is_som() helper function to check if a
  mirror contains SOM components
- Check in lod_primary_pick() to skip SOM mirrors when
  selecting primary mirror for writes
- Defensive checks in lod_declare_update_write_pending()
  to error out if non-stale SOM mirror is found

Tests 216-218 in sanity-flr.sh verify:
- SOM mirrors are never selected for writes
- Data mirrors are correctly selected as primary
- Write fails with ENODATA when only SOM mirror is
  non-stale
