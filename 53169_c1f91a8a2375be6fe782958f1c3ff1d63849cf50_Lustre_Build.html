PATCH:
LU-17295 llite: avoid VFS exclusive locking in atomic_open

This is an experimental patch to avoid the VFS exclusive locking
on parent directory in ->atomic_open during file creations.

When creating a file, a lock is not acquired during lookup_open.
This fix is being tried based on the expectation that LDLM
locking ensures sufficient consistency.
It is not clear whether this will work properly,
so further investigation is needed.
