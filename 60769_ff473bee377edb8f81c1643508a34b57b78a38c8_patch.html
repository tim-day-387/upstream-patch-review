LU-18934 osd: Add BRW IO latency statistics by size

Add new '{osd-ldiskfs,osd-zfs}.*.io_latency_stats' stats files, which
contain a percpu histogram of the server-side I/O latency by size.

These statistics can be used to identify performance issues related
to specific I/O sizes, such as small vs. large I/O operations.
Specifically, this can help identify whether small files are
causing performance issues due to high latency where they could be
masked by the creation of other large files.

These histograms are formatted the same as 'osc.*.io_latency_stats':
  io_latency_by_size:
  snapshot_time: 1756402276.152694808
  start_time:    14.921641461
  elapsed_time:  1756402261.231053347
  rd_4K: { 16us: 3, 32us: 3, 128us: 4, }
  rd_64K: { 32us: 5, 64us: 1, 128us: 1, 256us: 3, }
  rd_1024K: { 256us: 6, 1024us: 3, 2048us: 1, }
  rd_4096K: { 2048us: 9, 4096us: 1, }
  wr_4K: { 32us: 10, }
  wr_64K: { 64us: 8, 128us: 2, }
  wr_1024K: { 512us: 10, }
  wr_4096K: { 2048us: 10, }

The histograms are indexed by pages and store "binary usec" to avoid
64-bit divide for each stats increment.  The histograms are shown in
KiB and times are converted back to decimal usec for display.

Clean up the stats collection on the osc as well.
