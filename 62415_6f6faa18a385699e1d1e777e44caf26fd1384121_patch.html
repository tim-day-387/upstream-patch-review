LU-12187 lfs: improve the layout checks for ec

When checking the EC comps in a layout we need to take into account
also the number of stripes in the data comp it protects.
A problem here is that when we do these checks we do not yet
have the asosciation between the ec comp and the data comp it protects
so we can not use the (16 bits of timestampo is mirror_id)
But even if we had that the data comp has not yet had its mirror_id
assigned (it is still ==0 until the kernel is involved)

Maybe all these checks should be deferred and done inside the kernel
instead after it has assigned the mirror_ids ?
