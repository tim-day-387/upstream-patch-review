LU-18489 readahead: use page index when increase stride read window

We should use page index when increasing stride read window,
the read will be in units of page size anyway, plus currently
the increased window is calculated as
(ras_rpc_pages / stride_byte) * stride_length,
for a small stride_byte less than a page size this could end up
with a huge unintended readahead window.
