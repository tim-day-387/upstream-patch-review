LU-19572 utils: fix getdirstripe format

When 45c557ece518 updated the yaml format for `lfs getdirstripe`, it
changed the non-yaml format to always print out a header for the
directory components, even when no components were printed:

    $ lfs getdirstripe /mnt/lustre/test
    lmv_stripe_count: 0 lmv_stripe_offset: 0 lmv_hash_type: none
    mdtidx           FID[seq:oid:ver]

This output is confusing and is a change in behavior. This restores
the previous output:

    $ lfs getdirstripe /mnt/lustre/test
    lmv_stripe_count: 0 lmv_stripe_offset: 0 lmv_hash_type: none
