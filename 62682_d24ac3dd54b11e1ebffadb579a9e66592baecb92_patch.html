LU-19626 tests: multiop running out of single-letter commands

The multiop test utility uses single-letter
commands (a-z, A-Z) to perform file operations. We've used
up most of the alphabet - only 4 letters remain unused
(b, h, k, q). We need a way to expand beyond the current
52-command limit without breaking existing tests.

Implement two-character prefix commands using an unused
letter, 'q'. The prefix letter 'q' is followed by a
second character to specify the command (e.g., 'qe', 'qf').
This gives us ~50 new commands, and can expand if needed
by using q as a sub-prefix, eg, 'qqe'.

Implementation:
- Use 'q' as the first prefix letter
- Add 'qe[num]' for lseek(SEEK_END) with optional offset
to demonstrate
- Reserve 'qq' for future nested prefixes

Also fix a bug where commands with numeric arguments would
lose their return value due to the main loop resetting rc=0
at each iteration.

Add test_855 to sanity.sh to verify the new functionality.
