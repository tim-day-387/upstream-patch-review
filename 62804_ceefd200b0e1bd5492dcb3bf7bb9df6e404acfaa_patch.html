LU-0000 tests: add nosync flag test for SOM mirrors

SOM mirrors can have the nosync flag set to prevent
them from being updated during resync operations,
effectively creating a snapshot of the SOM metadata
at a point in time.

This patch adds test coverage in sanity-flr.sh
test_220 to verify that:
- The nosync flag can be set on SOM components
- SOM mirrors with nosync remain stale after resync
- Clearing nosync allows normal resync to proceed
- SOM mirror stale flag is cleared after resync

The test follows the same pattern as the parity
mirror nosync test in sanity-ec.sh test_6f.
