LU-12738 pfl: do not instantiate full PFL layout on append

Appending to a PFL file should avoid all component instantiation.
Let's lock only the region of already instantiated components and
check the size again once locked. If the position has changed and
the append size requires the next component instantiation, restart.
Thus, in the worst case we restart only the amount of component
times, not endlessly.

If a parallel IO writes to already instantiated components, it
does not differ from old code - extent lock conflict appears.

If a parallel IO writes to next components, i.e. there is no lock
conflicts, the resulted file will be like a sequential append and
a later write.
