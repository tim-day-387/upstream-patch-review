LU-0000 lod: block prefer flags on SOM components

SOM components should not have prefer flags
(LCME_FL_PREF_RW, LCME_FL_PREF_RD, LCME_FL_PREF_WR)
set on them, as they are not meant to be used for
normal I/O operations. SOM components contain only
metadata (file size information), not user data, so
directing I/O to them would be incorrect.

This patch adds:
- Server-side enforcement in lod_declare_layout_set()
  to reject setting prefer flags on SOM components
- Client-side validation in lfs_component_set() to
  provide user-friendly error messages
- Test coverage in sanity-flr.sh test_219

Test 219 verifies:
- Cannot set prefer/prefrd/prefwr on SOM components
- Can set prefer flags on data components
- Both client and server validation work correctly
