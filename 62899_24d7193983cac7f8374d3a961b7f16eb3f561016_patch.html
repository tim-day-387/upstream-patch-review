LU-19653 tests: fix multiop pop_arg initialization

The pop_arg() function uses a static variable cur_arg to
track the current argument position. The LU-15367 patch
changed this from being initialized once (static int
cur_arg = 3) to being reset on every call to pop_arg().

This causes an infinite loop when multiop tries to process
arguments, as cur_arg is reset to 3 or 4 on each call and
never advances past the first path argument. This results
in multiop printing usage messages repeatedly and filling
up test logs.

The fix ensures cur_arg is initialized only once by using
a static bool to track initialization state.
