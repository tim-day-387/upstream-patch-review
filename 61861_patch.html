LU-19485 lnet: Deprecate ip2nets and routes params

LNet currently supports network and routing configuration via kernel
module parameters (ip2nets= and routes=) supplied at module load time
(e.g. via modprobe.conf). This approach is being deprecated in favor
of Dynamic LNet Config (DLC) which uses user-space parsing of YAML +
netlink.

Modify the description of these parameters to note the deprecation,
and issue a one-time warning at startup if these parameters are being
used.

Surround relevant code in a version macro so the code will
automatically "disappear" in Lustre 2.18.
