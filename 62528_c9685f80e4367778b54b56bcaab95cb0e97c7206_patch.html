Fix for 6f

The issue is that reading the EC mirror gets truncated at EOF for the
file while the EC data might extend beyond.

Example:
File is 8 bytes.
Stripe size is 4M, we have two parities.

The parity data will be stored as:
First parity  EC offsets 0-8
Second parity EC offsets  - +8

We need to decide what "lfs mirror read" means if the mirorr we read is
a parity.
But for now, fold this patch into the commit that adds this test.
