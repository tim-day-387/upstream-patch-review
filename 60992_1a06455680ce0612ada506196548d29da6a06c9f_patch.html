LU-19274 contrib: Add spelling check to checkpatch-man

This patch adds:
- 'hunspell' check to checkpatch-man.pl.
- custom dictionary is reused from https://review.whamcloud.com/c/fs/lustre-release/+/62211
- custom dictionary is reused from /lustre-release/+/62211
  which is under contrib/scripts/exa.dic (TBD)
- -s option to see verbose spelling warnings. By default
  this is switch off.
- adds under summary count of spelling errors found
- if 'hunspell' binary is not found it skips spell check
  and reports "spelling check not done" under summary
- spell_dict is populated after reading file spelling-man.txt
  currently this is not being used.

Summary report with 'hunspell':
total: 0 errors, 2 warnings, 3 checks, 281 lines checked, 0 spelling errors

Summary report without 'hunspell':
total: 0 errors, 2 warnings, 3 checks, 281 lines checked, spelling check not done

This is first of the series, there would be 1 or 2 followup
patches which will be run individually on all Documentation/man*
folder and lustre key-word added to lustre_spell.en.pms
