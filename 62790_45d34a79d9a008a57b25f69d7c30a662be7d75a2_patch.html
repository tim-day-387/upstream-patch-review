LU-19478 nodemap: skip config distribution for ro commands

"lctl nodemap_test_nid" and "lctl nodemap_test_id" are nodemap
commands which do not modify the nodemap config and are therefore
read-only. However, if run from the MGS, "mgs_revoke_lock()" is called
which notifies all targets to fetch the current config from the MGS,
called nodemap synchronization.

When only persistent nodemaps are in use, this is an unnecessary
communication step. But, when dynamic nodemaps are set on MDSs or
OSSs, this results in wiping all dynamic nodemaps on the targets since
the MGS's nodemap config overwrites the in-memory nodemap on the
targets.

This patch skips nodemap synchronization for read-only nodemap
operations. Note, this patch does not skip synchronization for nodemap
commands that could update the nodemap config. For instance, two
consecutive "lctl nodemap_active 1" commands (within separate nodemap
synchronization steps) still results in nodemap synchronization.
