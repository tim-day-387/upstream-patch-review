LU-15367 tests: Fix iotrace_to_multiop.py parsing

Fix several bugs in the iotrace_to_multiop.py script
that prevented it from correctly parsing iotrace logs:

1. Parse flags as octal instead of hexadecimal, since
   iotrace logs show flags in octal format
2. Fix mmap idx parsing to handle additional fields in
   the iotrace output (idx=N vmf_flags=...)
3. Add do_file_read_iter and do_file_write_iter to
   FUNCTION_NAMES list to match actual iotrace output
4. Only parse START lines for read/write operations,
   not COMPLETED lines which show count=0

Also fix test_441a error checking to properly test
for success with [[ $ret == 0 ]] instead of the
incorrect [[ ret != 0 ]].
